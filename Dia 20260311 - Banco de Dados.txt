1) Qual a característica distingue um SGBD relacional de uma planilha Excel?

A - O SGBD possui esquema rígido com tipos, restrições e integridade referencial. --- Correto.
B - O SGBD armazena dados numéricos, a planilha suporta qualquer tipo de valor. --- Incorreto.
C - A planilha suporta transações ACID e o SGBD não garante consistência a falhas. --- Incorreto. (É o inverso)

2) O SalesDB usa MySQL em vez de MongoDB. Qual a justificativa técnica mais precisa para essa escolha?

A - MySQL suporta documents JSON com melhor desempenho que MongoDB. --- Incorreto
B - MongoDB não aceita queries; Acesso é apenas por chave primária. --- Incorreto
C - O SalesDB exige integridade referencial nativa entre pedidos e clientes_ garantida pela FK no modelo. --- Correto
D - MongoDB não possui suporte a consulta de agregação com SUM e COUNT. --- Incorreto


 DDL (Data Definition Language, ou Linguagem de Definição de Dados) é um subconjunto da SQL utilizado para definir, alterar e estruturar objetos de banco de dados, como tabelas, índices e views, em vez de manipular os dados em si. Principais comandos incluem CREATE, ALTER, DROP e TRUNCATE, operando na estrutura do banco.
 DML, ou Data Manipulation Language (Linguagem de Manipulação de Dados), é um subconjunto do SQL usado para gerenciar e alterar dados em bancos de dados relacionais. Seus comandos principais são INSERT (inserir), UPDATE (atualizar), DELETE (excluir) e, frequentemente, SELECT (consultar), focando na manipulação dos dados dentro das tabelas. 
 DQL (Data Query Language) é o subconjunto da linguagem SQL focado exclusivamente na consulta e recuperação de dados de bancos de dados relacionais. Seu principal e principal comando é o SELECT, utilizado para extrair informações específicas de tabelas, muitas vezes acompanhado de cláusulas como WHERE, ORDER BY e JOIN para refinar os resultados.
